# Tutor Scheduler Bot

Минималистичный Telegram-бот для репетитора английского языка. Помогает приглашать учеников, вести расписание занятий и автоматически отправлять напоминания в личные сообщения Telegram.

> Рабочее название проекта: **Tutor Scheduler Bot**. Его легко заменить на более брендовый вариант позже, например LessonPing или LessonFlow.

## Возможности

### Работа с учениками

- Первый пользователь `/start` становится владельцем-репетитором.
- Ученики подключаются только по персональным invite-ссылкам.
- Владелец не может использовать собственную invite-ссылку.
- Репетитор получает уведомление, когда ученик впервые подключился по invite-ссылке.
- Первый запуск ведет репетитора по короткому пути: добавить ученика → создать первое занятие.
- Ученика можно удалить вместе с его занятиями и расписанием.

### Планирование занятий

- `📅 Разовые занятия` — для одной или нескольких выбранных дат.
- `🔁 Постоянное расписание` — для занятий по дням недели или по выбранным конкретным датам.
- Можно выбрать одного, нескольких или всех учеников.
- Даты выбираются через inline-календарь с мультивыбором.
- Время выбирается кнопками или через `✏️ Другое время`.
- Перед созданием расписания по дням недели бот показывает предпросмотр.
- Во время мастера доступна кнопка `❌ Отмена`.

### Напоминания

- За 24 часа до занятия.
- За 1 час до занятия.
- Опционально за 10 минут.
- Напоминания отправляются ученикам в личные сообщения.

### Деплой и надежность

- SQLite без отдельного сервера.
- Готов к деплою на Railway (Worker-сервис).
- Локальный `.env` не попадает в GitHub.
- Базовые тесты покрывают ключевую бизнес-логику.

## Как пользоваться

1. Репетитор открывает бота и отправляет `/start`.
2. Бот регистрирует первого пользователя как владельца.
3. Репетитор нажимает `➕ Добавить ученика`.
4. Бот генерирует invite-ссылку.
5. Ученик открывает ссылку и привязывается к репетитору.
6. Репетитор получает уведомление и может сразу создать первое занятие.
7. Для одной или нескольких конкретных дат репетитор нажимает `📅 Разовые занятия`.
8. Для постоянного расписания нажимает `🔁 Постоянное расписание`.
9. Бот предлагает выбрать формат: `🗓 По дням недели` или `📅 Конкретные даты`.
10. Бот проводит по шагам: ученики, дни недели или конкретные даты, время, напоминания, создание.
11. После первого созданного занятия бот показывает завершение настройки.

Группа Telegram не нужна. Личные сообщения проще и их сложнее пропустить.

## Локальный запуск

Создайте бота через [@BotFather](https://t.me/BotFather) и скопируйте токен.

Создайте виртуальное окружение и установите зависимости:

```powershell
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

Создайте `.env` по примеру `.env.example`:

```env
BOT_TOKEN=your-token-from-botfather
BOT_TIMEZONE=Asia/Qyzylorda
BOT_DATABASE=reminders.sqlite3
REMINDER_POLL_SECONDS=60
BOT_PROXY=
```

Запустите бота из корня проекта:

```powershell
python -m bot.main
```

## Деплой на Railway

Бот работает через Telegram polling и не принимает HTTP-запросы, поэтому на Railway нужен **Worker-сервис** (не Web Service).

### Шаги

1. Создайте новый проект на [railway.com](https://railway.com) → **New Project → Deploy from GitHub repo**.
2. Выберите этот репозиторий.
3. Railway автоматически обнаружит `railway.toml` и запустит бота командой `python -m bot.main`.
4. Перейдите в **Variables** и задайте переменные окружения:

| Переменная | Значение |
|---|---|
| `BOT_TOKEN` | Токен от [@BotFather](https://t.me/BotFather) |
| `BOT_TIMEZONE` | `Asia/Qyzylorda` (или ваш часовой пояс) |
| `BOT_DATABASE` | `/data/reminders.sqlite3` |
| `REMINDER_POLL_SECONDS` | `60` |
| `BOT_PROXY` | Прокси-URL, если нужен (можно оставить пустым) |

### Персистентное хранилище для SQLite

Railway предоставляет примонтированный том (Volume) для хранения данных между перезапусками.

1. В настройках сервиса перейдите в **Volumes**.
2. Создайте Volume и задайте путь монтирования: `/data`.
3. Убедитесь, что переменная `BOT_DATABASE` указывает на `/data/reminders.sqlite3`.

> **Важно:** без Volume данные SQLite будут сбрасываться при каждом редеплое.

### Переменная BOT_TOKEN

Реальный токен нельзя коммитить в GitHub. Задайте его только через интерфейс Railway → Variables.
